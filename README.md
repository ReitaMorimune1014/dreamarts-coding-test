# 最長片道きっぷの旅 / Longest One-Way Ticket

株式会社ドリーム・アーツ 2028年新卒 エンジニア採用コーディングテスト

---

## 問題概要

駅と駅を結ぶ鉄道路線網において、同じ駅を2度通らずに移動できる最長経路を求める。

---

## 実行環境

- Node.js v18 以上
- OS: Windows / Mac / Linux (x86_64) 対応

---

## 実行方法

### PowerShell（Windows）

```powershell
Get-Content input.txt | node solution.js
```

### Git Bash / Mac / Linux

```bash
cat input.txt | node solution.js
```

### 入力フォーマット

```
始点のID, 終点のID, 距離
```

例:

```
1, 2, 8.54
2, 3, 3.11
3, 1, 2.19
3, 4, 4
4, 1, 1.4
```

### 出力例

```
1
2
3
4
```

## アルゴリズム

### 問題の性質

本問題は**最長単純路問題（Longest Simple Path Problem）**であり、一般グラフにおいてNP困難に分類される。

### 採用したアプローチ: DFS + バックトラック

全経路を深さ優先探索（DFS）で探索し、最長経路を求める。

```
1. 全駅を起点として試す
2. 起点からDFSで隣の駅へ進む
3. 行き止まり（訪問済み or 隣なし）に来たら、その時点の距離を記録
4. バックトラックで1つ前の駅に戻り、別ルートを試す
5. 全ルートを試し終えたら、記録の中で最長だった経路を出力する
```

### グラフの表現

鉄道路線は**双方向（無向グラフ）**としてモデル化し、`Map` による隣接リストで管理する。

```
駅1 → [{node: 2, dist: 8.54}, {node: 3, dist: 2.19}, {node: 4, dist: 1.4}]
駅2 → [{node: 1, dist: 8.54}, {node: 3, dist: 3.11}]
...
```

### 計算量

- 時間計算量: O(V! × V)（最悪ケース）
- 空間計算量: O(V)（再帰スタックおよび訪問済み管理）

実用的な路線網の規模では十分動作する。

---

## 設計上の判断

| 判断 | 理由 |
|------|------|
| 無向グラフとして扱う | 鉄道路線はA→BもB→Aも走行可能なため |
| `Map` で隣接リストを管理 | キーの型を保ちつつ `has` / `get` の意図が明確になるため |
| `Set` で訪問済みを管理 | 存在確認がO(1)で高速なため |
| `process.stdout.write` を使用 | 問題指定の改行コード `\r\n` を正確に出力するため |
